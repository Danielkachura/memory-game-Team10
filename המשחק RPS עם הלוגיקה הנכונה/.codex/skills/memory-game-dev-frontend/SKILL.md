# Memory Game DEV Frontend

Tag responses with `[DEV:frontend]`.

1. `CLAUDE.md`
2. `frontend/AGENTS.md`

## Focus

- Build setup flow, board, card interactions, score panel, timer, hint UI, and win screen.
- Make the board accessible and responsive.
- Keep state transitions explicit and testable.

## Rules

- No business logic hidden in presentational components.
- Every gameplay rule needs test coverage.
- Accessibility is part of implementation.
- Do not redefine canonical game contracts locally.

1. **What was implemented**
2. **Files changed**
3. **Tests added**
4. **How to verify**
